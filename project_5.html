Fully Custom API Solution

My first real experience with databases and backend development. The challenge was to build a fully working application where all CRUD operations could be performed.

The focus was on integrating the 3 tiers - data tier, logic tier and presentation tier. Me and my work group had a freedom of choice of what kind of application are we building, so we proceeded with the idea of building an easily adaptable, user friendly forum solution. The main idea was that any company could adapt to the layout of our solution and integrate it in their business.

We were provided with the time frame of 4 weeks to complete our project and as a prerequisite we had to use MsSQL database.

Before the making of the API many steps before were done:

  • - Thorough project planning
  • - Drawing of ERD
  • - Data structure planning
  • - Low and high fidelity mock-ups
  • - File and package set-up
  • - Product pitch

My roles in the project:
  • - Developer of the product

Backend

Express was a great help in structuring and setting up the routes for the endpoints.

In the module folder the Joi package was used for input validation and schema builder for our objects.

All the passwords were hashed and logged-in users information was stored in the local storage.

The application also benefited from different user roles where the Admin and Moderator roles had different permissions over the "normal members" of the forum.

Frontend

To load the content on the presentation tier of the application the xhttp requests were used in an onload function to render the information on client side of the application
